# Minimal RAG Query Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

**Goal:** Build a runnable first RAG demonstration that answers one fixed question from sample.pdf using keyword retrieval and DeepSeek.

**Architecture:** rag_query.py imports reading and chunking from rag_ingest.py, scores chunks by exact English keyword overlap, and sends the best chunks to ask_llm() in llm_client.py. It avoids Embedding models and vector databases. Reusable logic stays outside main() so local tests never call the API.

**Tech Stack:** Python 3.14, standard-library unittest and re, existing PyMuPDF, OpenAI-compatible DeepSeek client.

---

## File Structure

- Create: rag_query.py — keyword extraction, retrieval, Prompt construction, and fixed PDF demonstration.
- Create: tests/test_rag_query.py — offline retrieval and Prompt tests.
- Reuse: rag_ingest.py — read_text_file() and split_text().
- Reuse: llm_client.py — ask_llm().

### Task 2: Implement the Fixed RAG Demonstration

**Files:**
- Create: rag_query.py
- Test: tests/test_rag_query.py

- [ ] **Step 1: Create rag_query.py**

~~~python
import re

from llm_client import ask_llm
from rag_ingest import read_text_file, split_text


# 第一版固定使用这份 PDF，目的是先验证完整 RAG 流程。
DOCUMENT_PATH = "sample.pdf"

# 问题包含资料中的 Python 和 RAG，便于观察关键词检索如何选择片段。
QUESTION = "What does the document say about Python and RAG?"


def extract_keywords(text: str) -> set[str]:
    """提取英文和数字关键词，供第一版的字面匹配检索使用。"""
    # 转为小写，让 "Python" 与 "python" 被视为同一个关键词。
    normalized_text = text.lower()

    # 只提取连续的英文字母和数字；中文语义检索留给后续 Embedding 版本处理。
    return set(re.findall(r"[a-z0-9]+", normalized_text))


def retrieve_relevant_chunks(
    question: str,
    chunks: list[str],
    top_k: int = 2,
) -> list[str]:
    """按问题与片段的关键词重合数量，返回最相关的前几个片段。"""
    question_keywords = extract_keywords(question)
    scored_chunks: list[tuple[int, str]] = []

    for chunk in chunks:
        # 集合交集中的每个词都同时出现在问题和当前片段中。
        overlap_count = len(question_keywords & extract_keywords(chunk))
        scored_chunks.append((overlap_count, chunk))

    # 按得分从高到低排序；稳定排序会保留同分片段原有顺序。
    scored_chunks.sort(key=lambda item: item[0], reverse=True)

    # 即使所有得分都是 0，也保留片段，避免模型完全没有资料上下文。
    return [chunk for _, chunk in scored_chunks[:top_k]]


def build_rag_prompt(question: str, context_chunks: list[str]) -> str:
    """将检索片段和问题组织成要求模型基于资料回答的 Prompt。"""
    # 分隔线让模型更容易区分多个独立资料片段。
    context = "\n\n---\n\n".join(context_chunks)

    return f"""You are a helpful assistant. Answer the question using only the provided context.
If the context does not contain the answer, say that the context does not provide it.

Context:
{context}

Question:
{question}
"""


def main() -> None:
    """运行固定 PDF 与固定问题的最小 RAG 演示。"""
    document_text = read_text_file(DOCUMENT_PATH)

    # 200 个字符适合小示例；20 个重叠字符可减少边界截断信息的风险。
    chunks = split_text(document_text, chunk_size=200, chunk_overlap=20)

    if not chunks:
        raise ValueError("文档没有可检索的文本内容。")

    relevant_chunks = retrieve_relevant_chunks(QUESTION, chunks)

    print("检索到的资料片段：")
    for index, chunk in enumerate(relevant_chunks, start=1):
        print(f"\n--- 片段 {index} ---")
        print(chunk)

    prompt = build_rag_prompt(QUESTION, relevant_chunks)
    answer = ask_llm(prompt)

    print("\n模型回答：")
    print(answer)


if __name__ == "__main__":
    main()
~~~

Why: isolated functions are testable, while main() keeps the complete data flow visible.
