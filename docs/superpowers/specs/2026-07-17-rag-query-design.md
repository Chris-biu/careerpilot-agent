# 最小 RAG 问答设计

## 目标

新增 `rag_query.py`，用现有的文档读取、文本切分和 DeepSeek 客户端完成可运行的最小 RAG 问答演示。

第一版固定读取 `sample.pdf` 并使用固定问题，优先帮助初学者理解整个数据流。第二版在不改变核心逻辑的前提下，改为从终端接收文件路径和问题。

## 第一版流程

```text
sample.pdf
  -> read_text_file()
  -> split_text()
  -> 按关键词重合度为每个片段打分
  -> 选取得分最高的 1 到 2 个片段
  -> 片段 + 问题组成 Prompt
  -> ask_llm()
  -> 打印回答
```

## 模块职责

- `rag_ingest.py`：读取 TXT、Markdown、PDF，并将文本切成重叠片段。
- `llm_client.py`：读取环境变量中的 DeepSeek 配置并调用模型。
- `rag_query.py`：编排读取、切分、检索、构造 Prompt 和输出回答。

`rag_query.py` 不重复实现文件读取或模型调用，只导入并复用前两个模块的公开函数。

## 检索策略

第一版使用关键词重合度，不增加 Embedding 模型或向量数据库。

1. 将问题和文本片段转换为小写单词集合。
2. 计算问题关键词与每个片段关键词的交集数量。
3. 选择得分最高的 1 到 2 个片段作为上下文。

局限：它无法理解同义词或中英文概念映射；例如“检索增强生成”与“RAG”可能无法匹配。这是后续用 Embedding 向量检索升级的原因。

## Prompt 规则

向模型明确提供“资料片段”和“问题”，要求仅依据资料回答；如果资料不足，明确说明资料没有提供答案，而不是猜测。

## 错误处理

- 空文档：终止并提示没有可检索内容。
- 没有关键词匹配：仍保留得分最高的片段，避免没有上下文。
- 文件类型不支持：复用 `read_text_file()` 已有的 `ValueError`。
- API 配置缺失：复用 `ask_llm()` 已有的配置错误。

## 验证标准

运行 `python .\rag_query.py` 后：

1. 能读取 `sample.pdf`。
2. 能打印检索到的片段。
3. 能向 DeepSeek 提问并打印一段基于片段的回答。
4. 不新增数据库、Embedding 或向量检索依赖。
